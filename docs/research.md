# 电脑推荐配置系统：调研与需求初稿

## 目标与范围
- 目标：根据用户需求与预算档位，推荐台式机“主机”配置（全新，不含二手）。
- 覆盖场景：游戏、办公、编程/开发、设计/剪辑、AI 轻度实验等多场景。
- 地域：国内电商为主（京东/天猫/拼多多等），显卡以 AMD/NVIDIA 为主。

## 关键需求与假设（待确认）
- 预算档位：建议按价位段分档（示例：3k/5k/8k/1.2w/1.8w/2.5w+）。
- 设备范围：仅主机；默认不包含显示器/键鼠。
- 约束：不允许二手/矿卡；偏好全新零售渠道。
- 输出形式：前端可视化 + 配置清单 + 选型理由 + 预算占比。

## 场景维度与权重建议
- 游戏：GPU 优先，CPU 次之，内存/SSD 适中。
- 开发：CPU/内存优先，GPU 可选。
- 设计/剪辑：GPU + CPU + 内存均衡，SSD 适当提升。
- 轻度 AI：GPU 显存优先（NVIDIA/CUDA 生态），CPU 次之。

## 数据来源与采集思路（调研要点）
- 价格：国内电商价格（以 SKU 维度）。
- 性能：公开基准分数（CPU/GPU 性能排名/功耗）。
- 兼容性：主板/CPU/内存代际兼容，电源功耗估算。

候选数据源（需进一步核实可用性与授权）
- 电商平台商品列表与价格（可能需要手动/半自动维护）。
- 硬件参数库（CPU/GPU 规格、功耗、接口、代际）。
- 性能基准库（PassMark、3DMark、TechPowerUp 等公开排名）。

## 推荐规则框架（草案）
- 预算拆分：GPU/CPU/内存/存储/主板/电源/机箱比例。
- 兼容性规则：插槽/代际/功耗/尺寸。
- 场景权重：按场景调整部件预算占比。
- 品牌与禁忌：AMD/NVIDIA 选择 + 供应可得性。

## 风险与合规
- 数据来源的授权与更新频率。
- 电商价格波动导致推荐不稳定。
- 配置合理性（散热/功耗/空间）验证。

## 输出物（调研阶段）
- 数据源清单与可用性评估。
- 预算档位与场景权重确认。
- 初版推荐规则与验证用例。

## 需要你确认的问题
1. 预算档位如何划分？是否按“入门/主流/中高端/高端/旗舰”？
2. 需不需要把“显卡品牌偏好（AMD/NVIDIA）”做成用户可选项？
3. 是否需要支持“可升级方案”和“性价比优先/性能优先”模式？
4. 推荐结果是否需要“价格区间”而非精确价？
5. 有无已有的硬件清单/价格表，或需要我设计可维护的配置库格式？
